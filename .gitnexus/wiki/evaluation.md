# Evaluation

**Evaluation Module Documentation**

**Overview**
------------

The Evaluation module is responsible for evaluating the relevance and accuracy of search results in the context of legal texts. It uses natural language processing (NLP) techniques to analyze the search queries, retrieve relevant documents from a corpus, and determine the correctness of the retrieved documents.

**Purpose**
----------

The primary purpose of the Evaluation module is to provide a robust evaluation framework for the retrieval system, ensuring that it produces accurate and relevant results for legal searches. This involves assessing the performance of the retrieval algorithm in terms of precision, recall, and F1-score.

**How it Works**
----------------

The Evaluation module consists of two main components:

1.  **Query Analysis**: The `valutaCasi` function takes a search query as input and performs the following steps:
    *   Tokenization: The input query is tokenized into individual words.
    *   Embedding Generation: An embedding representation is generated for each token using an external language model (LLM).
    *   Corpus Retrieval: Relevant documents from a corpus are retrieved based on the generated embeddings.
2.  **Result Evaluation**: The `valutaCasi` function evaluates the retrieved documents and determines their correctness. This involves:
    *   Document Analysis: Each retrieved document is analyzed to determine its relevance and accuracy.
    *   Metric Calculation: Precision, recall, and F1-score are calculated for each document.

**Key Components**
------------------

*   **CasoValutazione**: Represents a single evaluation case, containing attributes such as the search query, document ID, and type of act.
*   **EsitoCasoValutazione**: Represents the outcome of an evaluation case, including attributes such as the retrieved article, first result, and response status.
*   **ReportValutazione**: Represents the overall report generated by the Evaluation module, containing metrics such as coverage, precision, and F1-score.

**Connection to the Rest of the Codebase**
-----------------------------------------

The Evaluation module interacts with other components in the following ways:

*   **Retrieval Module**: The `valutaCasi` function calls the `rispondiConFontiRag` and `cercaCorpusLegaleIbrido` functions from the Retrieval module to retrieve relevant documents.
*   **LLM Module**: The Evaluation module uses an external LLM to generate embeddings for each token in the search query.
*   **Test Suite**: The Evaluation module is tested using a test suite that exercises its functionality and ensures it produces accurate results.
